OEFF 2026 — Starter Project

Host venue communications onboarding

Scope: 6 hours/week through festival (~8 weeks, late February–mid April)
Working style: Kickoff call + async, then figure out a good meeting cadence
Rate: At the moment, we're able to offer $40/hr

The project

OEFF's host venue communications — webinar reminders, screening materials, check-ins — need to run on a reliable schedule as the festival approaches (April–May). We've recently built some new infrastructure: shared inboxes, templates, a communications plan. The role here is to step in and operate it, as well as document along the way.

We have host info sessions over the next few months and have just set up a mail merge campaign — looking to schedule regular reminders and continue to flesh out our process here.

Where we are: OEFF has moved from manual coordination to platform-based infrastructure — shared inboxes, a centralized host guide, webinar registration through Zoom. There's a communications plan, dependency maps, and email templates already in place. The work is to operate it, improve it, and write down what the next person would need to know.

What you'd work with

Operational tools (day one access):

  • Host Communications Plan — 5 communication streams with templates, schedule, and tool stack (Mailmeteor, Google Sheets, Gmail)
  • Dependency Map — prerequisites, approval chains, and rollback plans for every send
  • hosts@ shared inbox — the Gmail account host communications send from
  • Host Guide (hosts.oneearthfilmfest.org) — the persistent reference hosts are pointed to in every email

Background context (provided at kickoff):

  • Venue tier system — how OEFF classifies and supports different venue types
  • Team roster — who does what, who approves what
  • Contractor agreement — scope, budget, and team structure

How it works

  1. Week 1 — Kickoff + shadow — 30-min sync to walk through the comms plan, dependency map, and hosts@ inbox. I'll demo one send cycle end-to-end. Rest of the week: read the key docs, watch a live webinar reminder cycle, ask questions async.
  2. Weeks 2–3 — Take over Stream 1 — Own webinar reminders and recording follow-ups with me as backup. Follow the templates and the dependency map, keep a running doc of friction points and gaps.
  3. Weeks 4–7 — Expand and build — Pick up additional communication streams as the festival ramps up (deliverables shipping, host check-ins, data collection). Continue building operator notes as you go.
  4. Week 8 — Wrap up + debrief — Deliver operator notes and improved templates. 30-min sync to walk through what worked, what didn't, and what to hand off. Honest notes from doing the work, not a formal report.

Three things to track as you work

  1. What tripped you up? Where did the process assume context you didn't have? What required a question that should have been answered by the docs?
  2. What's missing from the handoff? If someone else needed to do this next year, what would you write down for them that doesn't exist yet?
  3. What could be tighter? Where do you see repeated patterns that could become templates, checklists, or standard sequences?

Broader work I'm interested in exploring

This starter project is a bite-sized way to work together and see how the fit feels. Beyond host comms, there are a few areas where I think collaboration could be valuable:

  • Operator guides — turning the running notes from this project into a repeatable playbook, so the next person doesn't start from scratch
  • Expanding to more communication streams — deliverables shipping, data collection forms, and host check-ins as the festival approaches
  • Filmmaker communications — similar infrastructure exists on the films@ side; natural next step once host comms are running
  • Process documentation — collaborative effort to capture how things actually work, so the systems outlast any one person
  • Festival week venue coordination — on-the-ground host support during April 22–28
  • Post-festival retrospective — a structured debrief while everything is still fresh

None of these are committed — they're possibilities that give shape to what a longer collaboration could look like. This is a project-based engagement, not a formal role. We're still figuring out what roles the org actually needs.

Festival week (April 22–28): Day-of venue coordination and on-site support is a separate engagement that we'll scope independently as the festival approaches.

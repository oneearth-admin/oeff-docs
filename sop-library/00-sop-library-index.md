## Pattern

Every SOP in this library follows the same structure:

```
1. Metadata block (owner, backup, tools, audience)
2. What this is (one paragraph)
3. The cycle / timeline (visual overview)
4. Step-by-step (numbered, one action per step)
5. Where everything lives (reference table)
6. Escalation (if/then pairs)
7. Anchor questions (what to track)
```

If a step involves sending, publishing, or committing — it names who owns the decision.

     |                          |                            |

**Erin handles:**
- Real-time presence (meetings, same-day questions)
- Contractor touchpoints (onboarding, check-ins)
- Video production and editing decisions
- Event capture logistics

**Garen handles:**
- Process design, documentation, specs
- Complex technical decisions
- Contractor scoping and SOW creation
- Escalation endpoint

**The handoff:**
- Garen creates the systems (templates, onboarding docs, escalation triggers)
- Erin operates within them (uses the templates, runs the onboarding, follows the triggers)
- Both update the shared dashboard so neither is chasing the other for status

## Scope Boundaries

### Included

All work described in Areas 1 and 2, including coordination with other OEFF contractors and the Technical Coordinator.

### Not included

- **Slide deck creation or venue-specific deck generation** — Erin may review decks for visual alignment and brand consistency, but deck production and assembly is not part of this scope
- **Sponsor logo compilation** — to be handled by intern or other support
- Technical specs (file formats, QC standards, equipment requirements)
- Process design or workflow architecture
- Contractor scoping or SOW creation
- Escalation decisions beyond documented triggers
- Equipment budgeting or vendor selection

### Scope change process

If new work arises beyond this scope, Erin will flag it before beginning. If it can be absorbed within the budget cap, a brief acknowledgment from Ana via text, email, or verbal confirmation is sufficient. If it requires additional budget, both parties will discuss before work begins.
